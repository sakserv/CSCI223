Github + Eclipse

How to setup Eclipse for use with Github.

1

Installing EGit

To install EGit, open Eclipse and navigate to the Eclipse Marketplace. The Eclipse Marketplace can be found by navigating to Help > Eclipse Marketplace. You should be presented with a dialogue similar to the one below (You should have an Install button versus the Update button seen below). Search for EGit and click install. Follow the onscreen dialogues using the defaults, you may need to accept the license agreement to proceed.

Restart Eclipse after installation.

Following the restart, validate the EGit was successfully installed by switching to the Git Repository Exploring. Navigate to Window > Open Perspective > Other. Validate that you have the perspective listed below.

EGit has been successfully installed. Proceed to the next step.

2

Setup Github Account and Repo

Follow the instructions in the attached document to setup a Github account and repo. Once the repo is created, copy the HTTPS GIT URL to your clipboard for use in the next step.

Download: GitHub_Installation.pdf

Proceed to the next step after creating the repo.

3

Clone the New Repo

Back in Eclipse, switch to the Git Repository Exploring (Window > Open Perspective > Other). Near the top of the window select Clone Repo.

Enter the URL captured from the previous step (Github Repo HTTPS URL) and your credentials. Hit next.

Verify the master branch is selected and hit next.

Select the local location where the Git repo will be stored. This is your working directory for local/offline commits. Click Finish.

You should now see the new repo listed under Git Repositories. Proceed to the next step.

4

Adding Projects to a Git Repo

Create an Eclipse project as you normally would (File > New > Static Web Project for example). You should now see the project listed in Project Explorer. Add some content to the project under the appropriate folder (WebContent in the case of a Static Web Project). Right click on the project and navigate to Team > Share Project as seen below.

On the dialogue that follows select Git.

On the next dialogue select the local repo you created in Step 3. Your selection should look similar to below. Click Finish.

In Project Explorer, your project should now have a tag following indicating it is under Git control, you should also notice quite a few new options under the Team menu.

You are now setup to use Git with Eclipse! The next step will take you through your first commit and push to upstream.

5

Committing Changes

You are now ready to make updates and commit those changes to Github. You will make code changes, add files, etc. as you normally would. Once code is tested and is ready to be checked in, right click on the project and navigate to Team >

Enter a commit comment in the large box at the top. Use the highlighted select all (or select the files you want to commit) and click on commit.

The new code has now been committed to your local git repo. When ready to push changes to Github, right click on the project and navigate to Team > Push to Upstream.

You have successfully committed code changes to Github!
